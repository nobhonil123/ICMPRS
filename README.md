## What Is This Repository?

Everything needed to reproduce the results in the paper above:

1. **The ICMPRS simulator** — Python code that generates realistic synthetic
   multimodal PD data calibrated to published Indian clinical norms, with
   formal MMD and KS distributional validity tests
2. **Three synthetic datasets** — 1,995 simulated participants (985 PD,
   1,010 healthy) across voice (25 features), handwriting (21 features),
   and gait (16 features) — **62 total feature dimensions**
3. **The Heterogeneous Ensemble** — RBF-SVM + Random Forest with:
   - **Adaptive Confidence Gating (ACG)** — signal-quality-aware dynamic fusion
   - **Cross-Modal Consistency Check (CMCC)** — clinical safety gate
4. **Evaluation scripts** — bootstrap CIs, McNemar's test, DeLong AUC,
   **Cultural Adaptation Gain (CAG)** metric, severity-stratified analysis
5. **Generalisability bound** — PAC-style formal worst-case guarantee
6. **Hardware benchmark** — latency and memory on ARM processor
7. **The IEEE LaTeX manuscript** — complete paper source

---

## ⚠️ Important: What This Is and Is Not

> **ICMPRS generates synthetic (computer-generated) data only.** All reported
> results are proof-of-concept on simulation data and do not constitute
> clinical validation. Real-world validation on collected Indian patient data
> under IRB approval is the essential next step. This repository is a
> reproducible research benchmark, not a deployable diagnostic tool.

## Quick Start (6 Steps)

### Step 1: Download this repository
Click the green **Code** button → **Download ZIP** → extract.

### Step 2: Install Python packages
```bash
pip install -r requirements.txt
```

### Step 3: (Optional) Regenerate the datasets
Pre-generated CSVs are in `data/`. To regenerate from scratch:
```bash
cd simulator
python generate_icmprs.py
```
This also runs formal MMD and KS distributional validity tests.

### Step 4: Train and evaluate the ensemble
```bash
cd models
python train_ensemble.py
```
Prints accuracy, sensitivity, AUC for ACG+CMCC and fixed-weight models.
Saves results to `results/performance_summary.csv`.

### Step 5: Run evaluation metrics
```bash
cd evaluation
python compute_metrics.py
python generalisability_bound.py
```

### Step 6: Run SHAP feature importance
```bash
cd evaluation
python shap_analysis.py
```

## Five Novel Indian-Specific Features

| Feature | Modality | What It Measures | Why It Matters for PD |
|---------|----------|-----------------|----------------------|
| **RCA** (Retroflex Consonant Articulation) | Voice | F₂ deviation during Hindi retroflex consonants | Tongue-curl motor act deteriorates early in PD |
| **NRE** (Nasalisation Residual Energy) | Voice | Nasal energy ratio, velopharyngeal leakage, nasal formant bandwidth | Hindi nasal vowels require velum control that PD impairs |
| **DPLD** (Devanagari Pen-Lift Dynamics) | Writing | Pen-lift count, duration, restart velocity | Devanagari shirorekha forces repeated lifts; PD makes them longer |
| **AJHI** (Akshar Junction Hesitation) | Writing | Conjunct transition time, pressure spike, inter-akshar pause | Devanagari conjuncts need complex motor sequencing; PD disrupts it |
| **BSV** (Barefoot Stride Variance) | Gait | Surface-normalised stride irregularity | Isolates neurological gait signal from terrain effects |

## Novel Algorithmic Contributions

### Adaptive Confidence Gating (ACG)
Replaces static fusion weights with signal-quality-aware dynamic gating:
```
w₁ = SQI_voice / (SQI_voice + SQI_move)
w₂ = 1 - w₁
P_final = w₁ · P_SVM + w₂ · P_RF
```
**Key result:** +8.8 pp accuracy gain under noisy conditions (0 dB SNR).

### Cross-Modal Consistency Check (CMCC)
Flags uncertain predictions for specialist referral:
```
CMCS = 1 - |P_SVM - P_RF|
If CMCS < 0.65 → "Refer to specialist" (not forced yes/no)
```
**Key result:** Routes 4.8% of cases; 72% are genuinely ambiguous.

### Cultural Adaptation Gain (CAG)
General-purpose metric for any cross-cultural medical AI:
```
CAG = (AUC_adapted - AUC_generic) / (AUC_generic - 0.5)
```
**Our result:** CAG = 0.138 (cultural features recover 13.8% of remaining headroom).

## Dataset Summary

All three CSVs share consistent `subject_id` and `label` columns.

| File | Rows | Feature Cols | Novel Features |
|------|------|-------------|----------------|
| `voice_final.csv` | 1,995 | 25 | RCA (1), NRE (3 sub-features) |
| `hand_final.csv` | 1,995 | 21 | DPLD (3 sub-features), AJHI (3 sub-features) |
| `gait_final.csv` | 1,995 | 16 | BSV (1) |
| **Total** | | **62** | **11 columns across 5 feature groups** |

**Label:** `0` = Healthy Control, `1` = Parkinson's Disease

**Additional columns:** `hy_stage` (H&Y severity: 0=HC, 1–3=PD),
`sex`, `dialect`, `device`
